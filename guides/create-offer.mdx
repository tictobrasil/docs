Uma oferta é um plano de precificação vinculado a um produto. Ela define quanto o cliente paga, como paga e em quais condições. Um mesmo produto pode ter múltiplas ofertas — por exemplo, um preço promocional para lançamento e um preço cheio permanente. Este guia cobre a criação de ofertas de **pagamento único**, onde o cliente é cobrado uma única vez no momento da compra.

## Criando uma oferta de pagamento único

<Steps>
  <Step title="Obtenha o hash do produto">
    Você precisa do `hash` do produto ao qual deseja vincular a oferta. Esse valor está disponível no campo `hash` da resposta ao [criar um produto](/guides/create-product) via `POST /products`, ou pode ser consultado na listagem de produtos pelo painel ou pela API.

    ```
    hash: "prd_a1b2c3d4e5f6"
    ```
  </Step>

  <Step title="Prepare o payload">
    Monte o corpo da requisição com o nome da oferta, o preço em centavos e o tipo de cobrança. Para pagamento único, use `charge_type: "one_time"`.

    ```json
    {
      "name": "Plano Básico",
      "price": 19700,
      "charge_type": "one_time"
    }
    ```

    O valor `19700` representa **R$ 197,00**. O preço mínimo aceito é `500` (R$ 5,00).
  </Step>

  <Step title="Envie a requisição">
    Faça um `POST` para `/api/v2/products/{hash}/offers`, substituindo `{hash}` pelo hash do produto. Inclua o `Idempotency-Key` para evitar duplicações em caso de retentativas.

    ```bash
    curl --request POST \
      --url https://api.ticto.com.br/api/v2/products/prd_a1b2c3d4e5f6/offers \
      --header "Authorization: Bearer ticto_live_..." \
      --header "Content-Type: application/json" \
      --header "Idempotency-Key: 7c9e6679-7425-40de-944b-e07fc1f90ae7" \
      --data '{
        "name": "Plano Básico",
        "price": 19700,
        "charge_type": "one_time"
      }'
    ```
  </Step>

  <Step title="Use o checkout_url">
    Uma resposta `201 Created` confirma a criação da oferta. O campo `checkout_url` já está pronto para ser compartilhado com seus clientes ou integrado ao seu site.

    ```json
    {
      "id": 308,
      "reference_id": "ofr_x9y8z7w6v5u4",
      "name": "Plano Básico",
      "price": 19700,
      "charge_type": "one_time",
      "status": "active",
      "checkout_url": "https://pay.ticto.com.br/checkout/ofr_x9y8z7w6v5u4"
    }
    ```

    Diferente dos produtos — que iniciam como `draft` — as ofertas já nascem com `status: "active"` e podem ser usadas imediatamente.
  </Step>
</Steps>

## Campos da oferta

<ParamField body="name" type="string" required>
  Nome da oferta exibido no checkout e nos relatórios. Use um nome descritivo, como "Plano Básico" ou "Oferta de Lançamento".
</ParamField>

<ParamField body="price" type="integer" required>
  Preço em centavos (inteiro). Mínimo: `500` (R$ 5,00). Exemplos: `19700` = R$ 197,00 · `9990` = R$ 99,90 · `50000` = R$ 500,00.
</ParamField>

<ParamField body="charge_type" type="string" required>
  Tipo de cobrança. Use `one_time` para pagamento único. Para assinaturas recorrentes, consulte o guia de [ofertas recorrentes](/guides/recurring-offers).
</ParamField>

<Tip>
  O campo `checkout_url` na resposta é o link de checkout pronto para compartilhar com seus clientes. Guarde-o na sua central após criar a oferta — você não precisará consultar a API novamente para obtê-lo.
</Tip>

## Métodos de pagamento padrão

Para produtos digitais (`course`, `ebook`, `software`), a Ticto configura automaticamente os métodos de pagamento mais utilizados no mercado brasileiro. Nenhuma configuração adicional é necessária:

| Método              | Configuração padrão         |
| ------------------- | --------------------------- |
| Cartão de crédito   | Parcelamento em até 12×     |
| PIX                 | Ativado automaticamente     |
| Boleto bancário     | Vencimento em 3 dias úteis  |

Esse comportamento é gerenciado pelo **Preset Resolver** da Ticto, que aplica as melhores práticas de checkout para maximizar sua conversão sem nenhuma configuração manual.

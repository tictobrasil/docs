---

Crie uma nova oferta vinculada a um produto da sua conta. Ofertas definem o preço, a forma de cobrança e o link de checkout que será disponibilizado ao comprador. Uma oferta pode ser de cobrança única (`one_time`) ou recorrente (`recurring`), com suporte a período de trial e número limitado de ciclos.

## Endpoint

```
POST /api/v2/products/{product_hash}/offers
```

## Headers obrigatórios

| Header            | Valor                        | Descrição                                                                                     |
|-------------------|------------------------------|-----------------------------------------------------------------------------------------------|
| `Authorization`   | `Bearer {seu_token}`         | Token de acesso da API.                                                                       |
| `Content-Type`    | `application/json`           | Formato do corpo da requisição.                                                               |
| `Idempotency-Key` | UUID v4 único por requisição | Garante que reenvios da mesma requisição não criem ofertas duplicadas em caso de falha de rede. |

## Corpo da requisição

<ParamField body="name" type="string" required>
  Nome comercial da oferta. Exibido na página de checkout para o comprador.
</ParamField>

<ParamField body="price" type="integer" required>
  Preço da oferta em centavos. Mínimo: `500` (equivalente a R$ 5,00). Exemplo: `9700` para R$ 97,00.
</ParamField>

<ParamField body="charge_type" type="string" required>
  Tipo de cobrança. Use `one_time` para cobrança única ou `recurring` para assinatura recorrente.
</ParamField>

<ParamField body="recurrence" type="object">
  Obrigatório quando `charge_type` é `recurring`. Define a periodicidade, período de trial e número de ciclos da assinatura.

  <Expandable title="Campos do objeto recurrence">
    <ParamField body="interval" type="string" required>
      Periodicidade da cobrança recorrente. Valores aceitos: `monthly`, `quarterly`, `semiannually`, `annually`.
    </ParamField>
    <ParamField body="trial_days" type="integer">
      Dias de acesso gratuito antes da primeira cobrança. Padrão: `0`.
    </ParamField>
    <ParamField body="cycles" type="integer">
      Número máximo de cobranças. Omita ou envie `null` para recorrência ilimitada.
    </ParamField>
  </Expandable>
</ParamField>

## Exemplos de requisição

<CodeGroup>

```bash Cobrança única
curl --request POST \
  --url "https://api.ticto.com.br/api/v2/products/PROD-ABC123/offers" \
  --header "Authorization: Bearer {seu_token}" \
  --header "Content-Type: application/json" \
  --header "Idempotency-Key: e4d909c2-90d6-4b85-a9e4-2a5dface7c3a" \
  --data '{
    "name": "Acesso Vitalício",
    "price": 29700,
    "charge_type": "one_time"
  }'
```

```bash Recorrente com trial
curl --request POST \
  --url "https://api.ticto.com.br/api/v2/products/PROD-ABC123/offers" \
  --header "Authorization: Bearer {seu_token}" \
  --header "Content-Type: application/json" \
  --header "Idempotency-Key: f7a12bc3-44e1-4cda-b09f-1e3d8c7f5a21" \
  --data '{
    "name": "Plano Anual",
    "price": 49700,
    "charge_type": "recurring",
    "recurrence": {
      "interval": "annually",
      "trial_days": 7,
      "cycles": null
    }
  }'
```

</CodeGroup>

## Resposta 201

A oferta é criada com sucesso e o objeto completo é retornado, incluindo o `id`, o `checkout_url` definitivo e a `created_at`.

```json Resposta 201
{
  "id": "OFRT-8H2K",
  "reference_id": 55190,
  "name": "Plano Anual",
  "price": 49700,
  "charge_type": "recurring",
  "recurrence": {
    "interval": "annually",
    "trial_days": 7,
    "cycles": null
  },
  "status": "active",
  "checkout_url": "https://checkout.ticto.com.br/OFRT-8H2K",
  "created_at": "2024-03-15T14:35:00Z"
}
```

<Note>
  O `checkout_url` retornado é o link definitivo de checkout para essa oferta. Armazene-o na sua central após a criação — ele não muda e pode ser usado diretamente em botões de compra ou campanhas de marketing.
</Note>

## Erros possíveis

| Status HTTP | Slug do erro           | Descrição                                                                                          |
|-------------|------------------------|----------------------------------------------------------------------------------------------------|
| `400`       | `bad_request`          | Corpo da requisição mal formatado ou JSON inválido.                                                |
| `401`       | `unauthenticated`      | Token ausente, inválido ou expirado.                                                               |
| `403`       | `forbidden`            | O token não possui o escopo `offers:write` necessário para esta rota.                              |
| `404`       | `product_not_found`    | Nenhum produto com o `product_hash` informado foi encontrado na conta.                             |
| `409`       | `conflict`             | Já existe uma oferta criada com o mesmo `Idempotency-Key`. A resposta retorna a oferta original.   |
| `422`       | `validation_error`     | Um ou mais campos do corpo não passaram na validação. Veja o array `details` para detalhes.        |
| `429`       | `too_many_requests`    | Limite de requisições atingido. Aguarde o tempo indicado no header `Retry-After`.                  |

Retorna os dados completos de uma oferta específica vinculada a um produto da sua conta. Use este endpoint para consultar o preço, tipo de cobrança, configurações de recorrência e o link de checkout de uma oferta individualmente.

## Endpoint

```
GET /api/v2/products/{product_hash}/offers/{offer_id}
```

## Parâmetros de path

<ParamField path="product_hash" type="string" required>
  Hash único do produto cadastrado na Ticto. Visível na URL do produto no painel.
</ParamField>

<ParamField path="offer_id" type="string" required>
  ID único da oferta no formato `OFRT-XXXX`. Retornado na criação ou na listagem de ofertas.
</ParamField>

## Requisição

```bash cURL
curl --request GET \
  --url "https://api.ticto.com.br/api/v2/products/PROD-ABC123/offers/OFRT-8H2K" \
  --header "Authorization: Bearer {seu_token}"
```

## Resposta 200

Exemplo de resposta para uma oferta de cobrança única (`one_time`). Nesse caso, o campo `recurrence` retorna `null`.

```json Resposta 200
{
  "id": "OFRT-8H2K",
  "reference_id": 55190,
  "name": "Acesso Vitalício",
  "price": 29700,
  "charge_type": "one_time",
  "recurrence": null,
  "status": "active",
  "checkout_url": "https://checkout.ticto.com.br/OFRT-8H2K",
  "created_at": "2024-03-15T14:35:00Z"
}
```

## Campos da resposta

<ResponseField name="id" type="string">
  Identificador único da oferta na Ticto, no formato `OFRT-XXXX`.
</ResponseField>

<ResponseField name="reference_id" type="integer">
  ID numérico interno da oferta no painel Ticto. Útil para reconciliação com sistemas legados ou relatórios exportados do painel.
</ResponseField>

<ResponseField name="name" type="string">
  Nome comercial da oferta exibido no checkout.
</ResponseField>

<ResponseField name="price" type="integer">
  Preço da oferta em centavos. Exemplo: `9700` equivale a R$ 97,00.
</ResponseField>

<ResponseField name="charge_type" type="string">
  Tipo de cobrança da oferta. Valores possíveis: `one_time` (cobrança única) ou `recurring` (assinatura recorrente).
</ResponseField>

<ResponseField name="recurrence" type="object | null">
  Configurações de recorrência da oferta. Presente somente quando `charge_type` é `recurring`. Retorna `null` para ofertas de cobrança única.

  <Expandable title="Campos do objeto recurrence">
    <ResponseField name="interval" type="string">
      Periodicidade da cobrança. Valores possíveis: `monthly`, `quarterly`, `semiannually`, `annually`.
    </ResponseField>
    <ResponseField name="trial_days" type="integer">
      Quantidade de dias de período de trial gratuito antes da primeira cobrança.
    </ResponseField>
    <ResponseField name="cycles" type="integer | null">
      Número máximo de cobranças. `null` indica recorrência ilimitada.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="status" type="string">
  Situação atual da oferta. Valores possíveis: `active` ou `inactive`.
</ResponseField>

<ResponseField name="checkout_url" type="string">
  URL direta de checkout para esta oferta. Pode ser compartilhada com clientes ou incorporada em botões de compra.
</ResponseField>

<ResponseField name="created_at" type="string">
  Data e hora de criação da oferta no formato ISO 8601. Exemplo: `2024-03-15T14:35:00Z`.
</ResponseField>

## Erros possíveis

| Status HTTP | Slug do erro        | Descrição                                                                         |
|-------------|---------------------|-----------------------------------------------------------------------------------|
| `401`       | `unauthenticated`   | Token ausente, inválido ou expirado.                                              |
| `403`       | `forbidden`         | O token não possui o escopo `offers:read` necessário para esta rota.              |
| `404`       | `offer_not_found`   | Nenhuma oferta com o `offer_id` informado foi encontrada para este produto.       |

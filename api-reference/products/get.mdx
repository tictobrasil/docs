Retorna os dados completos de um produto específico da sua conta, identificado pelo seu `hash`. O hash é obtido no momento da criação do produto ou através do endpoint de listagem.

## Endpoint

```
GET /api/v2/products/{product_hash}
```

## Parâmetros de path

<ParamField path="product_hash" type="string" required>
  O hash único do produto (obtido ao criar ou listar produtos). Exemplo: `abc123def`.
</ParamField>

## Requisição

```bash
curl --request GET \
  --url https://api.ticto.com.br/api/v2/products/abc123def \
  --header "Authorization: Bearer ticto_live_SEU_TOKEN_AQUI"
```

## Resposta

Retorna o objeto completo do produto correspondente ao hash informado.

```json
{
  "id": "PROD-K7M2",
  "hash": "abc123def",
  "name": "Curso de Marketing Digital",
  "type": "course",
  "description": "Aprenda marketing digital do zero.",
  "status": "active",
  "created_at": "2024-03-15T14:32:00Z"
}
```

## Campos da resposta

<ResponseField name="id" type="string">
  Identificador único do produto no formato `PROD-XXXX`.
</ResponseField>

<ResponseField name="hash" type="string">
  Hash único do produto, usado como referência nos demais endpoints da API.
</ResponseField>

<ResponseField name="name" type="string">
  Nome do produto cadastrado.
</ResponseField>

<ResponseField name="type" type="string">
  Tipo do produto. Valores possíveis: `course`, `ebook`, `subscription`, `software`, `physical`.
</ResponseField>

<ResponseField name="description" type="string | null">
  Descrição do produto, se preenchida; caso contrário, `null`.
</ResponseField>

<ResponseField name="status" type="string">
  Status atual do produto: `draft` (rascunho) ou `active` (ativo).
</ResponseField>

<ResponseField name="created_at" type="string">
  Data e hora de criação do produto no formato ISO 8601 (ex.: `2024-03-15T14:32:00Z`).
</ResponseField>

## Erros possíveis

| Código HTTP | Descrição |
|-------------|-----------|
| `401` | Token de autenticação ausente, inválido ou expirado. Verifique o header `Authorization`. |
| `404` | Produto não encontrado ou não acessível pela sua conta. |
| `429` | Limite de requisições excedido. Aguarde antes de realizar uma nova tentativa. |
